前端架构之js使用与插件DEMO

简介

此文档为下一版前端ui准备，由于样式借用了twitter bootstrap.css，所以不兼容ie6，请使用非IE6浏览器

整个前端架构组成：core.js，里面包含了Modernizr与yepnope.js和所有页面公共js代码

如何使用

一个网页面一般情况下只包含两个js文件，示例如下：

<!DOCTYPE html>
<html lang="zh-CN">
<head>
	<title></title>
	<meta charset="gbk">
	<!--加载核心脚本，里面包含了Modernizr与yepnope.js，还包含了每个页面共用的一些js代码，应当尽量精简之-->
	<script src="js/core.js"></script>
	<link rel="stylesheet" type="text/css" href="css/xxx.css" />
</head>
<body>
	<!--HTML代码开始-->
	<div>
		.........
	</div>
	<!--HTML代码结束-->
	<!--由于jQuery几乎每个页面都会要用到，所以没有必要做按需加载，这里直接在页面最底部引入之-->
	<script type="text/javascript" src="js/jquery-1.6.3.js"></script>
	<script>
	//你的业务逻辑脚本
	</script>
</body>
</html>

别名机制：几乎所有的UI组件，在core.js里都定义了一个别名，不需要在每个页面中引入它的具体路径，我们都通过异步动态加载需要的UI和功能组件，比如我要使用tab功能，代码如下： 

yepnope({
	load: alias.tabs,//引入tabs组件js文件
	callback: function (url, result, key) {
		$("ul.tabs").tabs("div.panes > div");//加载完成之后执行业务代码
	}
});
